Analysis

In the worst case, each step i requires a shift of i-1 elements for the insertion (consider an input list that is sorted in reverse order). Thus the runtime complexity is ùí™‚Å¢(n2). Even the optimization of using a binary search does not help us here, because the deciding factor in this case is the insertion. It is possible to use a data type with ùí™‚Å¢(log‚Å°n) insertion time, giving ùí™‚Å¢(n‚Å¢log‚Å°n) runtime, but then the algorithm can no longer be done as an in-place sorting algorithm. Such data structures are also quite complicated.

A similar algorithm to the insertion sort is the selection sort, which requires fewer data movements than the insertion sort, but requires more comparisons.
